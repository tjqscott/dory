# Dory — Changelog Skill

This project uses a `CHANGELOG.md` to maintain context across sessions.

**Reading:** When `CHANGELOG.md` is present in context, read it before responding. It contains a plain log of project actions and decisions.

**Writing:** When asked to update the changelog, append one or more plain lines summarising what was done in this session. No headings, no blank lines between entries, no markdown structure beyond bold or inline code where useful. Append only — never edit or remove existing entries.
